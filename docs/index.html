The 3pm problem

EADDRINUSE, solved.

Something's squatting on the port you need and you have no idea what. The usual answer is a detour through netstat, a PID, and Task Manager.

In PortPeek you type the port, see exactly which process owns it, and stop it. One click, back to work.

$ npm run dev
Error: listen EADDRINUSE: address already in use :::3000
    ...who even started that?
:3000 in use · vite · PID 14820 Free port
What you get

The whole picture for every port.

The context you actually need before you decide what to do about a port.

Full process context

PID, memory, uptime, executable, project directory, and the launch command for every listener.

Free a port in one click

Type a port number, see what holds it, stop the process. Reclaim :3000 without the terminal archaeology.

Network exposure, flagged

Instantly see when something's bound to 0.0.0.0 and reachable from other machines on your network.

Framework & app aware

Recognizes Postgres, Vite, Next.js, Docker, WSL and more, labelled with real icons instead of raw process names.

Filter the noise

Dev ports vs system ports, TCP and UDP, plus live search across ports, process names and PIDs.

Tiny and native

Built with Tauri, not Electron. ~5 MB, lives in your tray, and barely touches memory.
